Lección 11.4

Props opcionales con isset() en componentes Blade

Cómo pasar variables al layout sin que explote cuando la página no las define

📦 La analogía del formulario

Imagina un formulario con un campo "apodo" que es opcional. Si la persona no lo llena, el sistema no debería decir "ERROR: apodo no encontrado" — debería simplemente poner un valor por defecto o dejarlo vacío.

Con las variables en Blade layouts pasa lo mismo: si la vista no define $pageTitle, el layout no debería explotar. isset() es la forma de preguntar "¿existe esta variable antes de usarla?"

🔴 El error que teníamos

ErrorException
Undefined variable $pageTitle

Esto pasaba porque en app.blade.php teníamos:

<x-seo-meta title="{{ $pageTitle ?? 'Inicio' }}" />

El problema: las dobles llaves {{ }} evalúan la expresión PHP antes de pasarla. Si $pageTitle no existe como slot de Blade, PHP lanza el error antes de poder aplicar el ??.

📄 resources/views/layouts/app.blade.php — la corrección 
<!-- ❌ ROTO: {{ }} evalúa antes de que Blade sepa si el slot existe -->
<x-seo-meta title="{{ $pageTitle ?? 'Inicio' }}" />

<!-- ✅ CORRECTO: el : vincula directamente PHP con isset() seguro -->
<x-seo-meta
    :title="isset($pageTitle) ? (string)$pageTitle : 'Clínica Médica'"
    :description="isset($pageDescription) ? (string)$pageDescription : 'Atención médica especializada'"
/>

🔬 Línea por línea

Código¿Qué hace?
:title="..." El : delante del atributo significa "evalúa esto como PHP", no como texto
isset($pageTitle) Pregunta: "¿existe esta variable Y tiene un valor?" — si no existe, devuelve false sin error
? (string)$pageTitle Si existe, úsala y conviértela a string explícitamente
: 'Clínica Médica' Si NO existe, usa este valor por defecto

⚠️ Diferencia entre {{ }} y : en atributos Blade

title="{{ $var }}"

Evalúa $var inmediatamente como PHP dentro de comillas. Si $var no existe → error fatal antes de que puedas manejarlo.

:title="expr PHP"

Evalúa la expresión PHP completa. Puedes usar isset(), operadores ternarios y cualquier lógica sin que cause error.

🛠️ Cómo pasa una vista su título al layout

📄 resources/views/paginas/nosotros.blade.php 
<x-app-layout>
    <!-- Slot nombrado: el layout lo recibe como $pageTitle -->
    <x-slot name="pageTitle">Quiénes Somos — Clínica Médica</x-slot>
    <x-slot name="pageDescription">Conoce al equipo médico especializado que nos respalda.</x-slot>

    <div class="py-16 px-4">
        <h1>Nosotros</h1>
    </div>
</x-app-layout>
📄 resources/views/paginas/servicios.blade.php (sin definir el slot) 
<x-app-layout>
    <!-- Esta vista NO define pageTitle — el layout usa el valor por defecto -->
    <div class="py-16 px-4">
        <h1>Servicios</h1>
    </div>
</x-app-layout>

En el segundo caso, como $pageTitle no existe, el isset() devuelve false y el layout muestra "Clínica Médica" como título por defecto — sin ningún error.

📋 Resumen

  • Los slots nombrados (x-slot name="...") llegan al layout como variables PHP
  • Dentro de {{ }}, si la variable no existe → error. Usar : + isset() es la forma segura
  • La sintaxis :atributo="expr" en Blade = "evalúa esta expresión PHP como valor del atributo"
  • Las vistas que no necesiten un título personalizado simplemente no definen el slot
