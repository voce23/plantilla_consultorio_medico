Lección 10.2: Accesibilidad (a11y)

aria-labels, contraste, focus-visible, semantic HTML

🎯 Piensa en esto: Imagina que entras a un edificio. Las personas con discapacidad visual necesitan señales en braille, las personas en silla de ruedas necesitan rampas, y todas necesitan señales claras. En la web, la accesibilidad (a11y) son esas "rampas" digitales: permiten que personas con diferentes capacidades usen tu sitio sin problemas.
1 Skip to content link

Permite a usuarios de teclado saltar directamente al contenido, evitando navegar por todo el menú:

📍 Archivo: resources/views/layouts/app.blade.php

<body>
    {{-- Skip to main content link --}}
    <a href="#main-content" 
       class="sr-only focus:not-sr-only focus:absolute focus:top-4 
              focus:left-4 focus:z-50 focus:bg-medico-azul-600 
              focus:text-white focus:px-4 focus:py-2 focus:rounded-lg">
        Saltar al contenido principal
    </a>
    
    <!-- Navegación... -->
    
    <main id="main-content" tabindex="-1">
        {{ $slot }}
    </main>
</body>

💡 sr-only oculta visualmente pero los lectores de pantalla lo leen. Al hacer focus, se hace visible.

📝 ESTE ES EL CÓDIGO QUE VAMOS A APRENDER:

<!-- Botón con aria-label descriptivo -->
<button aria-label="Cerrar menú de navegación">
    <svg>...icono X...</svg>
</button>

<!-- Imagen con alt descriptivo -->
<img src="doctor.jpg" 
     alt="Dr. Juan Pérez, cardiólogo con 15 años de experiencia">

🔍 Vamos a entender línea por línea (como si fuera la primera vez):

aria-label="Cerrar menú de navegación"
→ Describe qué hace el botón para lectores de pantalla. Sin esto, solo dirían "botón" porque el contenido es un SVG.
alt="Dr. Juan Pérez..."
→ Texto alternativo que describe la imagen. Es obligatorio para accesibilidad y SEO. Nunca dejes alt vacío a menos que sea decorativa pura.
2 Contraste de colores

El contraste entre texto y fondo debe ser suficiente para personas con baja visión:

<!-- BUENO: Contraste alto (4.5:1 o más) -->
<p class="text-gray-900 bg-white">Texto oscuro sobre fondo claro</p>
<p class="text-white bg-medico-azul-700">Texto claro sobre fondo oscuro</p>

<!-- MALO: Contraste bajo (no usar) -->
<p class="text-gray-400 bg-gray-300">Texto gris claro sobre gris</p>

💡 Usa herramientas como WebAIM Contrast Checker para verificar. El mínimo es 4.5:1 para texto normal.

3 Focus visible

Los usuarios de teclado necesitan ver dónde está el foco:

<!-- Enlace con focus visible -->
<a href="#" class="focus:outline-none focus:ring-2 focus:ring-medico-azul-500 
                    focus:ring-offset-2 rounded">
    Enlace accesible
</a>

<!-- Botón con focus visible -->
<button class="bg-medico-azul-600 text-white px-4 py-2 rounded
               hover:bg-medico-azul-700
               focus:outline-none focus:ring-2 focus:ring-medico-azul-500 
               focus:ring-offset-2">
    Agendar cita
</button>

💡 focus:ring-2 crea un anillo alrededor del elemento. focus:ring-offset-2 deja espacio entre el anillo y el elemento.

4 HTML Semántico

Usa las etiquetas correctas para que los lectores de pantalla entiendan la estructura:

<!-- MALO: Todo con divs -->
<div class="header">...</div>
<div class="nav">...</div>
<div class="main">...</div>

<!-- BUENO: Etiquetas semánticas -->
<header>...</header>
<nav aria-label="Navegación principal">...</nav>
<main>
    <article>
        <h1>Título del artículo</h1>
        <section>
            <h2>Subtítulo</h2>
            <p>Contenido...</p>
        </section>
    </article>
</main>
<footer>...</footer>

💡 Las etiquetas semánticas (<header>, <nav>, <main>, <article>, <footer>) ayudan a los lectores de pantalla a navegar.

5 Formularios accesibles

Los formularios deben tener labels asociados correctamente:

<!-- BUENO: Label explícito con for -->
<label for="email">Correo electrónico</label>
<input type="email" id="email" name="email" required 
       aria-required="true" aria-describedby="email-help">
<small id="email-help">Usaremos tu email solo para confirmar la cita.</small>

<!-- BUENO: Label envolviendo el input -->
<label>
    Nombre completo
    <input type="text" name="nombre" required>
</label>

<!-- MALO: Sin label, solo placeholder -->
<input type="text" placeholder="Nombre"> {{-- NO HACER ESTO --}}

💡 aria-describedby conecta el input con texto adicional de ayuda. aria-required indica que es obligatorio.

6 Roles ARIA para componentes complejos

Para componentes interactivos como modales y menús desplegables:

<!-- Modal accesible -->
<div role="dialog" aria-modal="true" aria-labelledby="modal-titulo">
    <h2 id="modal-titulo">Confirmar cita</h2>
    <p>¿Deseas confirmar tu cita para el 15 de marzo?</p>
    <button>Confirmar</button>
    <button>Cancelar</button>
</div>

<!-- Menú desplegable -->
<button aria-expanded="false" aria-controls="menu-servicios">
    Servicios
</button>
<ul id="menu-servicios" role="menu" hidden>
    <li role="menuitem"><a href="#">Cardiología</a></li>
    <li role="menuitem"><a href="#">Dermatología</a></li>
</ul>

💡 role="dialog" indica que es un modal. aria-expanded indica si el menú está abierto o cerrado.

🎉 ¿Qué aprendimos hoy?

  • El link "Saltar al contenido" ayuda a usuarios de teclado
  • aria-label describe elementos que no tienen texto visible
  • El atributo alt es obligatorio en imágenes
  • El contraste mínimo es 4.5:1 para texto normal
  • focus:ring hace visible el foco del teclado
  • HTML semántico (<nav>, <main>, etc.) mejora la navegación
  • Los formularios siempre deben tener <label> asociado
