Lección 6.1: Modelo y Migración de Citas

Crear el modelo Cita con sus relaciones

🎯 Piensa en esto: Imagina que vas a un restaurante y haces una reserva. Necesitan anotar: tu nombre, para cuántas personas, qué día, qué hora y tu teléfono. Una cita médica es igual: necesitamos saber quién viene, con qué doctor, cuándo y por qué.
1 Crear el modelo con su migración

El comando make:model con la bandera -m crea tanto el modelo como la migración de base de datos.

📍 Terminal:

php artisan make:model Cita -m

💡 Esto crea dos archivos: app/Models/Cita.php y database/migrations/xxxx_create_citas_table.php

2 Definir la estructura de la migración

La migración define qué columnas tendrá la tabla en la base de datos. Es como dibujar el plano de una casa antes de construirla.

📍 Archivo: database/migrations/xxxx_create_citas_table.php

$table->id();
$table->foreignId('paciente_id')->constrained()->onDelete('cascade');
$table->foreignId('doctor_id')->constrained()->onDelete('cascade');
$table->foreignId('especialidad_id')->nullable()->constrained('especialidads')->onDelete('set null');
$table->date('fecha');
$table->time('hora');
$table->enum('estado', ['pendiente', 'confirmada', 'completada', 'cancelada'])->default('pendiente');
$table->text('motivo')->nullable();
$table->timestamps();

💡 foreignId crea una columna que referencia a otra tabla. onDelete('cascade') borra las citas si se borra el paciente.

📝 ESTE ES EL CÓDIGO QUE VAMOS A APRENDER:

📍 Archivo: app/Models/Cita.php

class Cita extends Model
{
    protected $fillable = [
        'paciente_id',
        'doctor_id', 
        'especialidad_id',
        'fecha',
        'hora',
        'estado',
        'motivo',
    ];

    protected $casts = [
        'fecha' => 'date',
    ];

    public function doctor(): BelongsTo
    {
        return $this->belongsTo(Doctor::class);
    }
}

🔍 Vamos a entender línea por línea:

protected $fillable = [...]
→ Lista de campos que se pueden llenar masivamente. Es como una lista de invitados: solo estos campos pueden entrar a la base de datos.
protected $casts = ['fecha' => 'date']
→ Le dice a Laravel: "cuando saques la fecha, conviértela a objeto Carbon". Así puedes hacer $cita->fecha->format('d/m/Y').
public function doctor(): BelongsTo
→ Define una relación "pertenece a". Una cita pertenece a un doctor. Laravel hace la magia de buscar en la tabla doctors.

📊 Estructura de la tabla citas:

CampoTipoDescripción
idbigintIdentificador único
paciente_idforeignIdReferencia al paciente
doctor_idforeignIdReferencia al doctor
fechadateFecha de la cita
horatimeHora de la cita
estadoenumpendiente/confirmada/completada/cancelada

👆 Los campos _id son claves foráneas que conectan con otras tablas.

📁 Código completo del modelo Cita 
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Relations\BelongsTo;

class Cita extends Model
{
    protected $fillable = [
        'paciente_id',
        'doctor_id',
        'especialidad_id',
        'servicio_id',
        'fecha',
        'hora',
        'estado',
        'motivo',
        'notas',
        'costo',
    ];

    protected $casts = [
        'fecha' => 'date',
        'costo' => 'decimal:2',
    ];

    public function paciente(): BelongsTo
    {
        return $this->belongsTo(Paciente::class);
    }

    public function doctor(): BelongsTo
    {
        return $this->belongsTo(Doctor::class);
    }

    public function especialidad(): BelongsTo
    {
        return $this->belongsTo(Especialidad::class);
    }
}

🎉 ¿Qué aprendimos hoy?

  • php artisan make:model Cita -m crea modelo + migración
  • $fillable protege contra asignación masiva no deseada
  • $casts convierte tipos de datos automáticamente
  • BelongsTo define relaciones "pertenece a"
  • foreignId()->constrained() crea claves foráneas con integridad referencial
