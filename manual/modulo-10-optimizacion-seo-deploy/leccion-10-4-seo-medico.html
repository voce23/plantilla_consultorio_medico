Lección 10.4: SEO Médico

Componente Blade SEO, Schema.org MedicalOrganization, Open Graph

🎯 Piensa en esto: Imagina que tienes el mejor restaurante del mundo, pero está en un callejón sin señales y sin nombre en la fachada. Nadie lo encontraría. El SEO es como poner letrero luminoso, buena dirección en Google Maps, y reseñas visibles: hace que los pacientes te encuentren cuando buscan "cardiólogo cerca de mí".
1 Componente x-seo-meta

Creamos un componente centralizado para manejar todas las meta tags desde un solo lugar:

📍 Archivo: resources/views/components/seo-meta.blade.php

@props([
    'title' => config('app.name', 'Clínica Salud'),
    'description' => 'Clínica médica especializada...',
    'keywords' => 'clínica médica, doctores, especialistas...',
    'image' => asset('images/og-default.jpg'),
    'canonical' => url()->current(),
    'type' => 'website',
])

💡 url()->current() obtiene automáticamente la URL actual. La URL canónica le dice a Google cuál es la versión original de la página.

📝 ESTE ES EL CÓDIGO QUE VAMOS A APRENDER:

📍 Archivo: resources/views/components/seo-meta.blade.php

<!-- Schema.org MedicalOrganization JSON-LD -->
<script type="application/ld+json">
{
    "@context": "https://schema.org",
    "@type": "MedicalOrganization",
    "name": "Clínica Salud",
    "description": "Clínica médica especializada...",
    "url": "https://clinicasalud.com",
    "telephone": "+52-123-456-7890",
    "address": {
        "@type": "PostalAddress",
        "streetAddress": "Av. Salud 123",
        "addressLocality": "Ciudad de México",
        "addressCountry": "MX"
    },
    "openingHoursSpecification": [{
        "@type": "OpeningHoursSpecification",
        "dayOfWeek": ["Monday","Tuesday","Wednesday","Thursday","Friday"],
        "opens": "08:00",
        "closes": "20:00"
    }]
}
</script>

🔍 Vamos a entender línea por línea (como si fuera la primera vez):

"@type": "MedicalOrganization"
→ Le dice a Google que es una organización médica. Google puede mostrar información especial en resultados (rich snippets) como horarios y teléfono.
type="application/ld+json"
→ Es el formato preferido por Google para datos estructurados. LD+JSON = Linked Data + JavaScript Object Notation.
"@context": "https://schema.org"
→ Indica el vocabulario estándar que usamos. Schema.org es el diccionario que Google, Bing y Yahoo entienden.
"openingHoursSpecification"
→ Google puede mostrar los horarios directamente en los resultados de búsqueda sin que el usuario entre al sitio.
2 Open Graph para redes sociales

Estas etiquetas controlan cómo se ve tu página cuando alguien la comparte en Facebook, WhatsApp o LinkedIn:

<!-- Open Graph básico -->
<meta property="og:type" content="website">
<meta property="og:url" content="{{ url()->current() }}">
<meta property="og:title" content="Dr. García - Cardiólogo | Clínica Salud">
<meta property="og:description" content="Más de 15 años de experiencia...">
<meta property="og:image" content="{{ asset('images/og-doctor-garcia.jpg') }}">
<meta property="og:locale" content="es_MX">

<!-- Twitter Card -->
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="Dr. García - Cardiólogo">
<meta name="twitter:description" content="Más de 15 años de experiencia...">
<meta name="twitter:image" content="{{ asset('images/og-doctor-garcia.jpg') }}">

💡 La imagen de Open Graph debe ser de al menos 1200x630px. Es lo que aparece cuando compartes el link.

3 Usar el componente en cada página

En el layout principal ya está incluido. Para personalizar por página, usa slots o variables:

📍 Archivo: Cualquier vista Blade

{{-- En la página de un doctor específico --}}
<x-app-layout>
    <x-slot name="pageTitle">Dr. {{ $doctor->nombre }} - {{ $doctor->especialidad }}</x-slot>
    <x-slot name="pageDescription">
        {{ $doctor->nombre }}, especialista en {{ $doctor->especialidad->nombre }}. 
        {{ $doctor->años_experiencia }} años de experiencia. Agenda tu cita.
    </x-slot>
    
    <!-- Contenido de la página -->
</x-app-layout>

💡 Pasar slots al layout te permite personalizar el SEO de cada página de forma independiente.

4 Schema.org para Doctor específico

Puedes agregar datos estructurados específicos para cada médico:

<script type="application/ld+json">
{
    "@context": "https://schema.org",
    "@type": "Physician",
    "name": "Dr. {{ $doctor->nombre }}",
    "image": "{{ asset('storage/' . $doctor->foto) }}",
    "url": "{{ route('doctores.show', $doctor) }}",
    "description": "{{ $doctor->bio }}",
    "medicalSpecialty": "{{ $doctor->especialidad->nombre }}",
    "worksFor": {
        "@type": "MedicalOrganization",
        "name": "{{ config('app.name') }}"
    },
    "availableService": {
        "@type": "MedicalProcedure",
        "name": "Consulta de {{ $doctor->especialidad->nombre }}"
    }
}
</script>

💡 @type: "Physician" le dice a Google que es un médico. Puede generar rich snippets con foto y especialidad.

5 Meta description efectiva

La description aparece en los resultados de Google. Debe tener entre 150-160 caracteres:

<!-- MALO: Muy genérico -->
<meta name="description" content="Clínica médica en México">

<!-- BUENO: Específico, con beneficio y llamada a la acción -->
<meta name="description" content="Clínica Salud en CDMX: cardiología, dermatología y 
pediatría con los mejores especialistas. Agenda tu cita en línea. +15 especialidades.">

<!-- MEJOR: Dinámico según la página -->
@php
$metaDesc = "Dr. {$doctor->nombre}: {$doctor->especialidad->nombre} con 
{$doctor->años_experiencia} años de experiencia. Consulta desde \${$doctor->precio_consulta}. 
Agenda en línea.";
@endphp
<meta name="description" content="{{ Str::limit($metaDesc, 160) }}">

💡 Str::limit($texto, 160) corta el texto a 160 caracteres con "..." si es más largo.

6 Validar los datos estructurados

Herramientas gratuitas para verificar que todo funciona:

1. Google Rich Results Test
   https://search.google.com/test/rich-results
   → Verifica si tus datos estructurados generan rich snippets

2. Schema.org Validator
   https://validator.schema.org
   → Valida la sintaxis del JSON-LD

3. Facebook Sharing Debugger
   https://developers.facebook.com/tools/debug/
   → Preview de cómo se ve al compartir en Facebook

4. Twitter Card Validator
   https://cards-dev.twitter.com/validator
   → Preview de Twitter Cards

💡 Siempre valida después de implementar. Un error de sintaxis en el JSON-LD puede hacer que Google lo ignore.

🎉 ¿Qué aprendimos hoy?

  • JSON-LD es el formato preferido por Google para datos estructurados
  • MedicalOrganization en Schema.org activa rich snippets con horarios
  • Open Graph controla la vista previa en redes sociales
  • La meta description debe tener 150-160 chars con un llamado a la acción
  • url()->current() genera la URL canónica automáticamente
  • Validar con Google Rich Results Test antes de publicar
