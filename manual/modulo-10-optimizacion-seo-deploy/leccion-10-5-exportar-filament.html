Lección 10.5: Exportar configuración Filament

Artisan command para backup y migración de recursos

🎯 Piensa en esto: Cuando un arquitecto termina de diseñar un edificio, guarda una copia de todos los planos: qué habitaciones hay, cómo están distribuidas, qué medidas tienen. Si algo se pierde o hay que construir otro edificio similar, tiene toda la información. Exportar la configuración de Filament es guardar esos "planos" de tu panel de administración.
1 El comando Artisan personalizado

Creamos un comando que analiza y exporta toda la configuración de Filament:

📍 Archivo: app/Console/Commands/FilamentExportConfig.php

namespace App\Console\Commands;

use Illuminate\Console\Command;

class FilamentExportConfig extends Command
{
    protected $signature = 'filament:export-config 
                            {--format=json : Formato de exportación (json, php)}
                            {--output=exports : Directorio de salida}';

    protected $description = 'Exporta la configuración y recursos de Filament';

    public function handle()
    {
        $this->info('🚀 Exportando configuración de Filament...');
        
        // Recopilar información
        $config = [
            'exportado_en' => now()->toDateTimeString(),
            'version_laravel' => app()->version(),
            'resources' => $this->exportResources(),
            'widgets' => $this->exportWidgets(),
            'panels' => $this->exportPanels(),
        ];
        
        // Guardar en JSON
        $filepath = storage_path("app/exports/filament-config.json");
        File::put($filepath, json_encode($config, JSON_PRETTY_PRINT));
        
        $this->info("✅ Exportado en: {$filepath}");
    }
}

💡 Los comandos Artisan se crean heredando de Command y definiendo $signature y handle().

📝 ESTE ES EL CÓDIGO QUE VAMOS A APRENDER:

📍 Terminal

# Exportar en formato JSON (por defecto)
php artisan filament:export-config

# Exportar en formato PHP
php artisan filament:export-config --format=php

# Exportar a una carpeta personalizada
php artisan filament:export-config --output=backups/filament

# Ver qué recursos existen
php artisan filament:export-config --format=json --output=exports

🔍 Vamos a entender línea por línea (como si fuera la primera vez):

protected $signature = 'filament:export-config {--format=json}'
→ Define el nombre del comando y sus opciones. --format=json es una opción con valor por defecto "json".
$this->info('mensaje')
→ Muestra un mensaje verde en la terminal. Es como echo pero con colores para la consola.
$this->option('format')
→ Lee el valor de la opción que pasó el usuario. Si ejecutas --format=php, retorna "php".
json_encode($config, JSON_PRETTY_PRINT)
→ Convierte el array PHP a texto JSON con sangría legible (no todo en una sola línea).
2 Qué exporta el comando

El comando recopila información de tres áreas:

    "exportado_en": "2026-03-10 17:30:00",
    "version_laravel": "12.0.0",
    "version_filament": "3.x",
    "resources": [
        {
            "nombre": "EspecialidadResource",
            "modelo": "App\\Models\\Especialidad",
            "label_singular": "Especialidad",
            "label_plural": "Especialidades",
            "campos_formulario": { "formulario_disponible": true },

    ],

        { "nombre": "StatsOverviewWidget" }
    ],

💡 Este JSON te dice exactamente qué Resources, Widgets y Panels tiene tu aplicación, útil para documentación o migración.

4 Script de deploy automático

El comando también genera un script con los comandos útiles para el servidor:

📍 Archivo generado: storage/app/exports/deploy-commands.sh

#!/bin/bash
# Comandos útiles para deploy de Filament

# 1. Limpiar cachés
php artisan cache:clear
php artisan config:clear

# 2. Optimizar para producción
php artisan config:cache
php artisan route:cache
php artisan view:cache

# 3. Filament específico
php artisan filament:upgrade
php artisan filament:cache-components

# 4. Storage
php artisan storage:link

# 5. Migraciones
php artisan migrate --force

💡 Este script se puede ejecutar con bash deploy-commands.sh en el servidor Linux.

5 Comandos Filament importantes

Comandos nativos de Filament que debes conocer:

# Ver todos los comandos de Filament
php artisan list filament

# Actualizar Filament (después de composer update)
php artisan filament:upgrade

# Cache de componentes (más rápido en producción)
php artisan filament:cache-components

# Limpiar cache de componentes (en desarrollo)
php artisan filament:clear-cached-components

# Generar un nuevo Resource
php artisan make:filament-resource NombreModelo --generate

# Generar un Widget
php artisan make:filament-widget NombreWidget --stats-overview

💡 filament:cache-components pre-renderiza los iconos y componentes, acelerando el panel admin.

🎉 ¿Qué aprendimos hoy?

  • Los comandos Artisan personalizados heredan de Command
  • $signature define el nombre y opciones del comando
  • $this->info(), $this->table() dan feedback en terminal
  • El comando exporta Resources, Widgets y Panels a JSON
  • filament:upgrade debe ejecutarse después de actualizar el paquete
  • filament:cache-components acelera el panel en producción
