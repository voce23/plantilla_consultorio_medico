Lección 11.1

Cómo funciona route() en Laravel

Registrar rutas nombradas en web.php y usarlas desde cualquier vista

🧭 Antes de empezar — la analogía

Imagina que cada página de tu sitio es un cuarto en un edificio. route() es el nombre de la puerta — no el número de piso ni la dirección exacta, sino un apodo que pusiste en el cartel.

Si el cuarto no tiene cartel (la ruta no está registrada), cuando alguien intente abrir esa puerta el edificio explota con error. Por eso en este módulo aprendemos a poner los carteles antes de usar los nombres.

📄 routes/web.php 
// Así se registra una ruta nombrada
Route::get('/nosotros', fn() => view('paginas.nosotros'))->name('nosotros');

// Y así se usa en cualquier vista Blade
<a href="{{ route('nosotros') }}">Nosotros</a>

🔬 Línea por línea

Código¿Qué hace?
Route::get('/nosotros', ...) Dice: "cuando alguien vaya a la URL /nosotros, responde con esto"
fn() => view('paginas.nosotros') Una función corta que devuelve la vista resources/views/paginas/nosotros.blade.php
->name('nosotros') Le pone un nombre/apodo a esa ruta — puedes inventar el nombre que quieras
route('nosotros') En la vista, esto genera automáticamente la URL /nosotros sin que la escribas tú

🛠️ Todas las rutas que agregamos

Esto es el web.php completo con todas las rutas públicas del proyecto:

📄 routes/web.php 
<?php

use Illuminate\Support\Facades\Route;

// ── Páginas públicas ──────────────────────────
Route::get('/', fn() => view('welcome'))->name('inicio');
Route::get('/nosotros', fn() => view('paginas.nosotros'))->name('nosotros');
Route::get('/servicios', fn() => view('paginas.servicios'))->name('servicios');
Route::get('/contacto', fn() => view('paginas.contacto'))->name('contacto');

// ── Páginas legales ───────────────────────────
Route::get('/politica-de-privacidad', fn() => view('paginas.politica-privacidad'))
    ->name('politica-privacidad');
Route::get('/terminos-de-uso', fn() => view('paginas.terminos-uso'))
    ->name('terminos-uso');

// ── Especialidades ────────────────────────────
Route::get('/especialidades', fn() => view('paginas.especialidades'))
    ->name('especialidades.index');
Route::get('/especialidades/{slug}', fn($slug) => view('paginas.especialidad-detalle', ['slug' => $slug]))
    ->name('especialidades.show');

// ── Médicos ───────────────────────────────────
Route::get('/nuestros-medicos', fn() => view('paginas.doctores'))
    ->name('doctores.index');
Route::get('/nuestros-medicos/{slug}', fn($slug) => view('paginas.doctor-detalle', ['slug' => $slug]))
    ->name('doctores.show');

// ── Blog ──────────────────────────────────────
Route::get('/blog', fn() => view('paginas.blog'))->name('blog.index');
Route::get('/blog/{slug}', fn($slug) => view('paginas.blog-detalle', ['slug' => $slug]))
    ->name('blog.show');

// ── Citas ─────────────────────────────────────
Route::get('/reservar-cita', fn() => view('paginas.reservar-cita'))
    ->name('citas.agendar');

// ── Dashboard (con autenticación) ─────────────
Route::middleware(['auth:sanctum', config('jetstream.auth_session'), 'verified'])
    ->group(function () {
        Route::get('/dashboard', fn() => view('dashboard'))->name('dashboard');
    });

💡 ¿Por qué usar route('nombre') y no /nosotros directo?

❌ URL directa (frágil)

<a href="/nosotros">

Si cambias la URL en web.php, tienes que buscar y reemplazar en TODOS los archivos.

✅ route() (resistente)

route('nosotros')

Si cambias /nosotros a /acerca-de, el enlace se actualiza solo en toda la app.

📋 Resumen de lo que aprendiste

  • Una ruta nombrada = URL + nombre personalizado con ->name()
  • En Blade, route('nombre') genera la URL automáticamente
  • Si la ruta no existe, el sitio explota con "Route not found" al cargar la página
  • El footer y nav llaman a rutas — todas deben estar registradas en web.php
