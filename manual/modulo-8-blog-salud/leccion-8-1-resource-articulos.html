Lección 8.1: Resource Artículos en Filament

CRUD completo con MarkdownEditor, relaciones y SEO

🎯 Piensa en esto: Imagina que tienes una revista de salud. Necesitas: secciones (categorías), artículos escritos por doctores, fotos llamativas, y que todo esté organizado para que los lectores encuentren lo que buscan. Eso es exactamente lo que vamos a crear digitalmente.
1 Crear los modelos y migraciones

Necesitamos dos tablas: categorías (para organizar) y artículos (el contenido). Fíjate que un artículo pertenece a una categoría.

📍 Terminal:

php artisan make:model Categoria -m
php artisan make:model Articulo -m

💡 Creamos ambos modelos porque un artículo siempre va a tener una categoría.

2 Definir la relación en las migraciones

La clave está en categoria_id - eso conecta cada artículo con su categoría.

📍 Archivo: database/migrations/2026_03_10_210547_create_articulos_table.php

$table->foreignId('categoria_id')
    ->constrained('categorias')
    ->onDelete('cascade');

💡 onDelete('cascade') significa que si borras una categoría, todos sus artículos se borran también. Evita artículos huérfanos.

3 Crear el Resource de Categorías primero

Siempre crea primero las categorías, porque cuando crees un artículo necesitarás seleccionar una categoría existente.

📍 Terminal:

php artisan make:filament-resource Categoria --generate

💡 El --generate crea el Form y Table automáticamente.

📝 ESTE ES EL CÓDIGO QUE VAMOS A APRENDER:

📍 Archivo: app/Filament/Resources/Articulos/Schemas/ArticuloForm.php

MarkdownEditor::make('contenido')
    ->label('Contenido del Artículo')
    ->required()
    ->fileAttachmentsDirectory('blog/contenido')
    ->columnSpanFull(),

Select::make('categoria_id')
    ->label('Categoría')
    ->relationship('categoria', 'nombre')
    ->searchable()
    ->preload()
    ->required()
    ->createOptionForm([
        TextInput::make('nombre')
            ->required()
            ->maxLength(100),
    ])

🔍 Vamos a entender línea por línea (como si fuera la primera vez):

MarkdownEditor::make('contenido')
→ Un editor de texto enriquecido que usa Markdown (sintaxis simple con # para títulos, * para listas, etc.). Es como escribir en Word pero con códigos sencillos.
fileAttachmentsDirectory('blog/contenido')
→ Permite arrastrar y soltar imágenes directamente en el editor. Las guarda automáticamente en la carpeta que le indicas.
Select::make('categoria_id')
→ Crea un menú desplegable para elegir la categoría. Guardamos el ID de la categoría, no su nombre.
relationship('categoria', 'nombre')
→ Le dice a Filament: "Este campo está relacionado con el modelo Categoria, y muestra el campo 'nombre' en el desplegable".
createOptionForm([...])
→ ¡Aquí está lo bonito! Si la categoría que necesitas no existe, puedes crearla al vuelo sin salir del formulario.
4 El campo SEO para Google

Para que tu artículo aparezca en Google, necesitas meta descripción y keywords.

📍 Archivo: app/Filament/Resources/Articulos/Schemas/ArticuloForm.php

Section::make('SEO (Opcional)')
    ->description('Optimización para motores de búsqueda')
    ->collapsible()
    ->schema([
        TextInput::make('meta_descripcion')
            ->label('Meta Descripción')
            ->maxLength(160)
            ->helperText('Máximo 160 caracteres. Aparece en Google.'),
    ])

💡 collapsible() hace que esta sección se pueda contraer, ocupando menos espacio visual.

📊 Así se ve la relación entre tablas:

Tabla: categorias
id: 1 nombre: Nutrición color: #22c55e
id: 2 nombre: Cardiología color: #ef4444

⬇️ Relación: hasMany ⬇️

Tabla: articulos
id: 1 titulo: 5 consejos... categoria_id: 1 → Pertenece a Nutrición
id: 2 titulo: Cuida tu corazón categoria_id: 2 → Pertenece a Cardiología

👆 La columna destacada conecta ambas tablas.

📁 ¿Quieres ver el código completo del formulario? (opcional)

Esto es solo para que veas dónde va tu código. No necesitas memorizar todo esto:

<?php

namespace App\Filament\Resources\Articulos\Schemas;

use Filament\Forms\Components\DateTimePicker;
use Filament\Forms\Components\FileUpload;
use Filament\Forms\Components\MarkdownEditor;
use Filament\Forms\Components\Select;
use Filament\Forms\Components\Textarea;
use Filament\Forms\Components\TextInput;
use Filament\Forms\Components\Toggle;
use Filament\Schemas\Components\Section;
use Filament\Schemas\Schema;
use Illuminate\Support\Str;

class ArticuloForm
{
    public static function configure(Schema $schema): Schema
    {
        return $schema
            ->components([
                Section::make('Información Principal')
                    ->schema([
                        TextInput::make('titulo')
                            ->required()
                            ->live(onBlur: true)
                            ->afterStateUpdated(fn (callable $set, $state) => 
                                $set('slug', Str::slug($state))
                            ),

                        Select::make('categoria_id')
                            ->relationship('categoria', 'nombre')
                            ->searchable()
                            ->preload()
                            ->required()
                            ->createOptionForm([
                                TextInput::make('nombre')->required(),
                            ]),

                        FileUpload::make('imagen_destacada')
                            ->image()
                            ->directory('blog/imagenes'),
                    ])
                    ->columns(2),

                Section::make('Contenido')
                    ->schema([
                        MarkdownEditor::make('contenido')
                            ->required()
                            ->fileAttachmentsDirectory('blog/contenido')
                            ->columnSpanFull(),
                    ]),
            ]);
    }
}

🎉 ¿Qué aprendimos hoy?

  • Creamos dos modelos relacionados: Categoría (uno) y Artículo (muchos)
  • foreignId + constrained crea la relación en la base de datos
  • MarkdownEditor permite escribir contenido enriquecido con imágenes
  • createOptionForm permite crear categorías sin salir del formulario de artículo
  • Los campos SEO ayudan a que Google encuentre y muestre tus artículos
