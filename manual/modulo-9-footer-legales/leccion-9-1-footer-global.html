Lección 9.1: Footer Global con Jetstream

Layout component con 4 columnas: logo, links, especialidades y contacto

🎯 Piensa en esto: Imagina que entras a un edificio de oficinas. En el lobby siempre hay un directorio que te dice qué hay en cada piso, cómo contactar a las empresas, y dónde están las salidas de emergencia. El footer es eso: el directorio de tu sitio web que está presente en todas las páginas.
1 Crear el componente Footer

Vamos a crear un componente Blade reutilizable que se incluirá en el layout principal.

📍 Archivo: resources/views/components/footer.blade.php

# Creamos el archivo directamente

💡 Los componentes en views/components se pueden usar con la sintaxis <x-footer />.

2 Consultar especialidades dinámicamente

El footer muestra las especialidades desde la base de datos, así siempre está actualizado.

📍 Archivo: resources/views/components/footer.blade.php

@php
use App\Models\Especialidad;

$especialidades = Especialidad::activas()
    ->ordenadas()
    ->limit(6)
    ->get();
@endphp

💡 Usamos limit(6) para no saturar el footer si hay muchas especialidades.

📝 ESTE ES EL CÓDIGO QUE VAMOS A APRENDER:

📍 Archivo: resources/views/components/footer.blade.php

<footer class="bg-medico-azul-900 text-white">
    <div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-4 gap-12">
        
        <!-- Columna 1: Logo -->
        <div>
            <!-- Logo y descripción -->
            <x-redes-sociales />
        </div>

        <!-- Columna 2: Links Rápidos -->
        <div>
            <h3>Links Rápidos</h3>
            <ul>...</ul>
        </div>

        <!-- Columna 3: Especialidades -->
        <div>
            <h3>Especialidades</h3>
            <ul>
                @foreach($especialidades as $esp)
                    <li>{{ $esp->nombre }}</li>
                @endforeach
            </ul>
        </div>

        <!-- Columna 4: Contacto -->
        <div>
            <h3>Contacto</h3>
            <!-- Dirección, teléfono, email -->
        </div>
    </div>
</footer>

🔍 Vamos a entender línea por línea (como si fuera la primera vez):

class="bg-medico-azul-900 text-white"
→ Fondo azul oscuro (el más intenso de nuestra paleta) con texto blanco. Es el estándar para footers profesionales.
grid-cols-1 md:grid-cols-2 lg:grid-cols-4
→ Responsive: en móvil es 1 columna, en tablet 2 columnas, en desktop 4 columnas. Se adapta a cualquier pantalla.
<x-redes-sociales />
→ Incluimos el componente de redes sociales que creamos. Así no repetimos código.
@foreach($especialidades as $esp)
→ Recorremos las especialidades de la base de datos y creamos un link para cada una.
3 La barra inferior (Copyright)

Siempre incluye la barra de copyright y links a páginas legales.

📍 Archivo: resources/views/components/footer.blade.php

<div class="border-t border-medico-azul-800">
    <div class="flex flex-col md:flex-row justify-between">
        <p>© {{ date('Y') }} Clínica Salud</p>
        <div class="flex gap-6">
            <a href="{{ route('politica-privacidad') }}">Privacidad</a>
            <a href="{{ route('terminos-uso') }}">Términos</a>
        </div>
    </div>
</div>

💡 date('Y') muestra el año actual automáticamente. Nunca se te olvida actualizarlo.

4 Incluir el footer en el layout

El footer debe aparecer en todas las páginas, así que lo incluimos en el layout principal.

📍 Archivo: resources/views/layouts/app.blade.php

<body>
    <!-- Header -->
    <header>...</header>
    
    <!-- Contenido principal -->
    <main>{{ $slot }}</main>
    
    <!-- Footer -->
    <x-footer />
</body>

💡 {{ $slot }} es donde se inyecta el contenido de cada página.

🎉 ¿Qué aprendimos hoy?

  • El footer es un componente reutilizable que va en todas las páginas
  • Las 4 columnas organizan: branding, navegación, especialidades y contacto
  • Consultamos especialidades de la BD para mantenerlo actualizado automáticamente
  • La barra inferior incluye copyright dinámico y links legales obligatorios
  • El diseño responsive se adapta a móvil, tablet y desktop
