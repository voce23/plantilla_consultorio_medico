Lección 11.3

Corregir JSON-LD en Blade — el error con @type

Por qué Blade "se confunde" con el JSON y cómo generarlo de forma segura con PHP

🤔 La analogía del cocinero despistado

Imagina que le dices a tu cocinero: "prepara esto exactamente como dice la receta". Pero hay una trampa: el cocinero tiene la costumbre de interpretar todo lo que empieza con @ — lo ve como una instrucción suya.

Si la receta dice @context, él cree que es una instrucción de cocina y se confunde. Eso mismo le pasa a Blade con el JSON-LD: ve el @ y piensa que es una directiva suya como @if o @foreach.

🔴 El error que obtuvimos

ParseError
syntax error, unexpected end of file, expecting "elseif" or "else" or "endif"
resources/views/components/seo-meta.blade.php:69

Causa: había JSON-LD escrito directamente en el archivo Blade, con estas partes problemáticas:

  • "@context" — Blade lo lee como directiva @context (no existe → error)
  • "@type" — igual, Blade cree que es @type (no existe → error)
  • {{ $variable }} dentro del JSON — Blade lo procesa pero puede generar caracteres inválidos en el JSON
❌ ANTES — seo-meta.blade.php (código roto) 
<script type="application/ld+json">
{
    "@context": "https://schema.org",
    "@type": "MedicalOrganization",
    "name": "{{ config('app.name') }}",
    "description": "{{ $description }}",
    "url": "{{ url('/') }}"
}
</script>
✅ DESPUÉS — seo-meta.blade.php (código corregido) 
@php
    $schemaOrg = json_encode([
        '@context'    => 'https://schema.org',
        '@type'       => 'MedicalOrganization',
        'name'        => config('app.name'),
        'description' => $description,
        'url'         => url('/'),
        'address'     => [
            '@type'           => 'PostalAddress',
            'streetAddress'   => 'Av. Principal 123',
            'addressLocality' => 'Ciudad de México',
            'addressCountry'  => 'MX',
        ],
        'openingHours' => ['Mo-Fr 08:00-20:00', 'Sa 09:00-14:00'],
    ], JSON_UNESCAPED_SLASHES | JSON_UNESCAPED_UNICODE | JSON_PRETTY_PRINT);
@endphp

<script type="application/ld+json">{!! $schemaOrg !!}</script>

🔬 Línea por línea

Código¿Qué hace?
@php ... @endphp Le dice a Blade: "ejecuta PHP puro aquí, no interpoles nada"
json_encode([...]) PHP convierte el array a JSON válido automáticamente
'@context' => '...' Al estar dentro de un array PHP, el @ no es una directiva Blade — es texto normal
JSON_UNESCAPED_SLASHES Evita que las URLs queden como "https:\/\/..." (con barras raras)
JSON_UNESCAPED_UNICODE Permite que los acentos y ñ queden legibles en vez de \u00f3
{!! $schemaOrg !!} Imprime el JSON sin escapar los caracteres (necesario para que sea JSON válido)

🏆 Regla de oro para JSON en Blade

NUNCA escribas JSON directamente en Blade si contiene @ o variables {{ }}.

Siempre usa:
1. @php → construye un array PHP
2. json_encode() → convierte a JSON
3. {!! $variable !!} → imprime sin escape

⚠️ ¿Por qué {!! !!} y no {{ }}?

Sintaxis¿Qué hace con caracteres especiales?¿Cuándo usar?
{{ $var }} Convierte " en &quot;, < en &lt; — protege de XSS Texto normal visible al usuario
{!! $var !!} Imprime exactamente como está, sin modificar nada JSON generado por PHP (ya es seguro)

📋 Resumen

  • Blade interpreta todo @ como una directiva PHP — incluso dentro de <script>
  • JSON-LD con "@context" y "@type" causa ParseError si se escribe directamente
  • La solución: construir el JSON con json_encode() dentro de @php
  • Imprimir con {!! !!} para no escapar las comillas del JSON
