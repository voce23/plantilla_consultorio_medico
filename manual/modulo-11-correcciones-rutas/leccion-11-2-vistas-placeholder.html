Lección 11.2

Vistas placeholder para rutas pendientes

Cómo crear páginas básicas para que el sitio no explote mientras terminamos de construir

🏗️ La analogía del edificio en construcción

Cuando construyen un edificio, ponen carteles que dicen "próximamente" o "en construcción" antes de terminar cada piso. Así la gente sabe que existe pero que aún no está listo.

En Laravel pasa lo mismo: si registras una ruta pero no creas la vista, el sitio explota con error. La solución es crear una vista mínima que diga "esta sección viene pronto".

🔴 El error que teníamos

El footer y el menú llamaban a rutas como route('nosotros'), route('blog.index'), etc. pero:

  1. Esas rutas no estaban en web.php → RouteNotFoundException
  2. Aunque las pusiéramos, las vistas no existían → InvalidArgumentException: View not found

El resultado: el sitio explotaba en cualquier página que tuviera el footer.

📄 resources/views/paginas/nosotros.blade.php (vista placeholder) 
<x-app-layout>
    <div class="max-w-4xl mx-auto py-16 px-4 text-center">
        <h1 class="text-3xl font-bold text-medico-azul-800 mb-4">
            Nosotros
        </h1>
        <p class="text-gray-500">Esta sección está en construcción. Vuelve pronto.</p>
    </div>
</x-app-layout>

📄 resources/views/paginas/especialidad-detalle.blade.php (con parámetro dinámico) 
<x-app-layout>
    <div class="max-w-4xl mx-auto py-16 px-4 text-center">
        <h1 class="text-3xl font-bold text-medico-azul-800 mb-4">
            Especialidad: {{ $slug }}
        </h1>
        <p class="text-gray-500">Detalle de esta especialidad próximamente.</p>
        <a href="{{ route('especialidades.index') }}"
           class="mt-6 inline-block text-medico-azul-600 hover:underline">
            ← Ver todas las especialidades
        </a>
    </div>
</x-app-layout>

🔬 ¿Qué es {slug} en la ruta?

Parte¿Qué significa?
{slug} en web.php Un "comodín" que captura lo que ponga el usuario en la URL
URL: /especialidades/cardiologia $slug tendrá el valor "cardiologia"
['slug' => $slug] Pasamos ese valor a la vista como variable $slug
{{ $slug }} en la vista Muestra el valor que llegó de la URL

📋 Resumen

  • Cada ruta necesita su vista — sin ella, error fatal
  • Las vistas placeholder usan <x-app-layout> para heredar header + footer
  • Las rutas con {slug} reciben un parámetro dinámico de la URL
  • Este enfoque "en construcción" evita errores mientras desarrollamos sección a sección
