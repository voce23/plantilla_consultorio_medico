Lección 7.1: Resource Testimonios en Filament

CRUD completo para gestionar testimonios de pacientes

🎯 Piensa en esto: Imagina que tienes un buzón de sugerencias en tu clínica donde los pacientes dejan notas contando su experiencia. Algunas notas son tan buenas que quieres mostrarlas en la pared para que otros pacientes las vean. Este Resource es como tener ese buzón organizado digitalmente, donde puedes aprobar qué testimonios se muestran y cuáles no.
1 Crear el modelo y la migración

Primero necesitamos la "ficha" donde guardaremos cada testimonio. Fíjate que guardamos: quién lo escribió, su foto, cuántas estrellas nos dio, y el comentario.

📍 Terminal:

php artisan make:model Testimonio -m

💡 El -m es mágico: crea el modelo Y la migración al mismo tiempo.

2 Definir los campos en la migración

Aquí le decimos a la base de datos qué información guardar de cada testimonio. Mira, incluimos campos para aprobar o destacar testimonios.

📍 Archivo: database/migrations/2026_03_10_205301_create_testimonios_table.php

$table->string('nombre');
$table->string('foto')->nullable();
$table->tinyInteger('estrellas')->default(5);
$table->text('comentario');
$table->string('servicio')->nullable();
$table->boolean('aprobado')->default(false);
$table->boolean('destacado')->default(false);

💡 aprobado nos sirve para moderar: solo los testimonios aprobados se ven en la web.

3 Crear el Filament Resource

Ahora generamos todo el panel de administración automáticamente. Filament hace el trabajo pesado por nosotros.

📍 Terminal:

php artisan make:filament-resource Testimonio --generate

💡 Cuando te pregunte por el "title attribute", escribe nombre.

📝 ESTE ES EL CÓDIGO QUE VAMOS A APRENDER:

📍 Archivo: app/Filament/Resources/Testimonios/Schemas/TestimonioForm.php

Section::make('Información del Paciente')
    ->schema([
        TextInput::make('nombre')
            ->label('Nombre completo')
            ->required(),

        FileUpload::make('foto')
            ->label('Foto de perfil')
            ->image()
            ->directory('testimonios/fotos'),

        Select::make('estrellas')
            ->label('Calificación')
            ->options([
                1 => '⭐ 1 estrella',
                2 => '⭐⭐ 2 estrellas',
                3 => '⭐⭐⭐ 3 estrellas',
                4 => '⭐⭐⭐⭐ 4 estrellas',
                5 => '⭐⭐⭐⭐⭐ 5 estrellas',
            ])
            ->default(5),

        Toggle::make('aprobado')
            ->label('Aprobado para publicar'),
    ])

🔍 Vamos a entender línea por línea (como si fuera la primera vez):

Section::make('Información del Paciente')
→ Creamos una caja con título que agrupa campos relacionados. Es como una sección en un formulario de papel.
TextInput::make('nombre')
→ Un campo de texto simple para que escriban el nombre del paciente.
FileUpload::make('foto')->image()
→ Un botón para subir archivos que solo acepta imágenes. Fíjate que le decimos que guarde en la carpeta testimonios/fotos.
Select::make('estrellas')->options([...])
→ Un menú desplegable donde elijen de 1 a 5 estrellas. Lo bonito es que mostramos las estrellas como emojis ⭐ para que sea visual.
Toggle::make('aprobado')
→ Un interruptor ON/OFF para decidir si este testimonio se muestra en la web o sigue pendiente de revisión.

📊 Así se ve tu tabla en la base de datos:

id nombre foto estrellas comentario aprobado
1 María García testimonios/fotos/maria.jpg 5 Excelente atención, muy profesional... true
2 Carlos López null 4 Buen servicio, recomendado false

👆 Los campos destacados son los que más usaremos para mostrar en la web.

📁 ¿Quieres ver el código completo del formulario? (opcional)

Esto es solo para que veas dónde va tu código. No necesitas memorizar todo esto:

<?php

namespace App\Filament\Resources\Testimonios\Schemas;

use Filament\Forms\Components\FileUpload;
use Filament\Forms\Components\Select;
use Filament\Forms\Components\Textarea;
use Filament\Forms\Components\TextInput;
use Filament\Forms\Components\Toggle;
use Filament\Schemas\Components\Section;
use Filament\Schemas\Schema;

class TestimonioForm
{
    public static function configure(Schema $schema): Schema
    {
        return $schema
            ->components([
                Section::make('Información del Paciente')
                    ->schema([
                        TextInput::make('nombre')
                            ->label('Nombre completo')
                            ->placeholder('Ej: María García')
                            ->required()
                            ->maxLength(100)
                            ->columnSpan(2),

                        FileUpload::make('foto')
                            ->label('Foto de perfil')
                            ->image()
                            ->imageEditor()
                            ->directory('testimonios/fotos')
                            ->maxSize(1024)
                            ->columnSpan(1),
                    ])
                    ->columns(3),

                Section::make('Valoración')
                    ->schema([
                        Select::make('estrellas')
                            ->label('Calificación')
                            ->options([
                                1 => '⭐ 1 estrella',
                                2 => '⭐⭐ 2 estrellas',
                                3 => '⭐⭐⭐ 3 estrellas',
                                4 => '⭐⭐⭐⭐ 4 estrellas',
                                5 => '⭐⭐⭐⭐⭐ 5 estrellas',
                            ])
                            ->default(5)
                            ->required(),

                        TextInput::make('servicio')
                            ->label('Servicio recibido')
                            ->placeholder('Ej: Consulta de Cardiología'),
                    ])
                    ->columns(2),

                Section::make('Testimonio')
                    ->schema([
                        Textarea::make('comentario')
                            ->label('Comentario')
                            ->required()
                            ->rows(4)
                            ->maxLength(1000),
                    ]),

                Section::make('Publicación')
                    ->schema([
                        Toggle::make('aprobado')
                            ->label('Aprobado para publicar')
                            ->default(false),

                        Toggle::make('destacado')
                            ->label('Destacar testimonio')
                            ->default(false),
                    ])
                    ->columns(2),
            ]);
    }
}

🎉 ¿Qué aprendimos hoy?

  • Creamos un modelo Testimonio con todos los campos necesarios (nombre, foto, estrellas, comentario)
  • Usamos FileUpload para que los administradores suban fotos de los pacientes
  • El campo Select con emojis hace que elegir estrellas sea visual y amigable
  • Los toggles de "aprobado" y "destacado" nos dan control total sobre qué se muestra en la web
  • Organizamos el formulario en secciones para que sea más fácil de usar
