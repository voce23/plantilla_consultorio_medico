Lección 6.2: Formulario de Citas en Filament

Crear el Resource de Citas con formulario completo

🎯 Piensa en esto: Es como tener una recepcionista digital. En lugar de llamar por teléfono para agendar, el administrador entra al panel, selecciona paciente, doctor, fecha y ¡listo! Filament hace todo el trabajo pesado.
1 Crear el Resource de Citas

Filament puede generar automáticamente el resource con todos sus componentes.

📍 Terminal:

php artisan make:filament-resource Cita --generate

💡 El flag --generate crea el formulario y tabla automáticamente basándose en el modelo.

2 Personalizar el CitaResource

Agregamos etiquetas en español y configuramos el grupo de navegación.

📍 Archivo: app/Filament/Resources/Citas/CitaResource.php

protected static ?string $modelLabel = 'cita';
protected static ?string $pluralModelLabel = 'citas';
protected static string|UnitEnum|null $navigationGroup = 'Gestión de Citas';
protected static ?int $navigationSort = 1;
3 Organizar el formulario en secciones

Usamos Section para agrupar campos relacionados, como separar carpetas en un archivero.

📍 Archivo: app/Filament/Resources/Citas/Schemas/CitaForm.php

Section::make('Información del Paciente')
    ->description('Datos del paciente')
    ->icon('heroicon-o-user')
    ->schema([
        Select::make('paciente_id')
            ->relationship('paciente', 'nombre_completo')
            ->searchable()
            ->preload()
            ->required(),
    ])

📝 ESTE ES EL CÓDIGO QUE VAMOS A APRENDER:

📍 Archivo: app/Filament/Resources/Citas/Schemas/CitaForm.php

Select::make('doctor_id')
    ->label('Doctor')
    ->relationship('doctor', 'nombre_completo', 
        fn ($query) => $query->where('activo', true))
    ->searchable()
    ->preload()
    ->required()
    ->live()
    ->afterStateUpdated(function (callable $set) {
        $set('especialidad_id', null);
        $set('servicio_id', null);
    })

🔍 Vamos a entender línea por línea:

->relationship('doctor', 'nombre_completo', fn ($query) => ...)
→ Carga doctores mostrando su nombre completo, pero filtra solo los activos. El fn es una función flecha de PHP.
->live()
→ Hace que el campo sea "en vivo". Cuando cambia, Filament reacciona inmediatamente sin recargar la página.
->afterStateUpdated(function (callable $set) {...})
→ Ejecuta código después de cambiar el valor. Aquí limpiamos especialidad y servicio cuando cambia el doctor.

📁 Código completo del formulario 
Section::make('Detalles de la Cita')
    ->icon('heroicon-o-stethoscope')
    ->schema([
        Select::make('doctor_id')
            ->relationship('doctor', 'nombre_completo', 
                fn ($query) => $query->where('activo', true))
            ->searchable()
            ->preload()
            ->required()
            ->live()
            ->afterStateUpdated(fn (callable $set) => $set('especialidad_id', null)),
        
        Select::make('servicio_id')
            ->relationship('servicio', 'nombre')
            ->searchable()
            ->preload()
            ->live()
            ->afterStateUpdated(function ($state, callable $set) {
                if ($state) {
                    $servicio = \App\Models\Servicio::find($state);
                    $set('costo', $servicio?->precio);
                }
            }),
    ])
    ->columns(2)

🎉 ¿Qué aprendimos hoy?

  • make:filament-resource genera el CRUD completo
  • Section::make() agrupa campos visualmente
  • ->live() habilita reactividad en tiempo real
  • ->afterStateUpdated() ejecuta lógica al cambiar valores
  • ->relationship() conecta con modelos relacionados
