Lección 7.3: Contador de Estadísticas

Componente StatsCounter con animación de números

🎯 Piensa en esto: Imagina que estás viendo un documental sobre la naturaleza y aparece en pantalla "Más de 1 millón de especies en el Amazonas". El número no aparece de golpe, sino que va subiendo rápidamente desde cero hasta llegar al millón. Ese efecto visual capta tu atención y hace que el dato sea más impactante. Eso es exactamente lo que vamos a hacer.
1 Crear el componente Blade

Vamos a crear un componente reutilizable que muestre estadísticas con animación. Este es diferente porque no necesita Livewire, es puro Blade + Alpine.js.

📍 Archivo: resources/views/components/stats-counter.blade.php

# No hay comando, creamos el archivo directamente

💡 Los componentes Blade son más ligeros que Livewire cuando no necesitas interactividad compleja.

2 Definir las props del componente

Queremos que el componente sea flexible: que podamos pasarle diferentes estadísticas según la página donde lo usemos.

📍 Archivo: resources/views/components/stats-counter.blade.php

@props([
    'stats' => [],
    'titulo' => null,
])

@php
$statsDefault = [
    ['valor' => 15000, 'sufijo' => '+', 'label' => 'Pacientes Atendidos', 'icono' => 'users'],
    ['valor' => 15, 'sufijo' => '+', 'label' => 'Años de Experiencia', 'icono' => 'calendar'],
    ['valor' => 25, 'sufijo' => '+', 'label' => 'Especialidades', 'icono' => 'academic-cap'],
    ['valor' => 98, 'sufijo' => '%', 'label' => 'Satisfacción', 'icono' => 'heart'],
];
$stats = !empty($stats) ? $stats : $statsDefault;
@endphp

💡 @props define qué parámetros puede recibir el componente. Si no le pasas nada, usa valores por defecto.

📝 ESTE ES EL CÓDIGO QUE VAMOS A APRENDER:

📍 Archivo: resources/views/components/stats-counter.blade.php

<div 
    x-data="{ current: 0, target: {{ $stat['valor'] }}, duration: 2000, startTime: null }"
    x-init="
        $watch('startAnimation', (value) => {
            if (value) {
                const animate = (timestamp) => {
                    if (!startTime) startTime = timestamp;
                    const progress = Math.min((timestamp - startTime) / duration, 1);
                    const easeOut = 1 - Math.pow(1 - progress, 3);
                    current = Math.floor(easeOut * target);
                    if (progress < 1) {
                        requestAnimationFrame(animate);
                    }
                };
                requestAnimationFrame(animate);
            }
        })
    "
>
    <span x-text="current">0</span>{{ $stat['sufijo'] }}
</div>

🔍 Vamos a entender línea por línea (como si fuera la primera vez):

x-data="{ current: 0, target: 15000, duration: 2000, startTime: null }"
→ Creamos variables en Alpine: current es el número que se muestra (empieza en 0), target es el número final, duration es cuánto dura la animación (2 segundos), y startTime guarda cuándo empezó.
$watch('startAnimation', (value) => { ... })
→ Alpine "observa" la variable startAnimation. Cuando cambia a true, empieza la animación. Esto permite que el contador arranque cuando el usuario hace scroll y ve la sección.
const animate = (timestamp) => { ... }
→ Creamos una función que se ejecuta en cada frame de la animación. timestamp es el tiempo actual en milisegundos.
const progress = Math.min((timestamp - startTime) / duration, 1)
→ Calculamos qué tan avanzada va la animación. Va de 0 (inicio) a 1 (final). Math.min asegura que no pase de 1.
const easeOut = 1 - Math.pow(1 - progress, 3)
→ Aquí está la magia: aplicamos una "curva de suavizado". El número empieza rápido y va frenando al final, como cuando un coche frena suavemente. Sin esto, la animación se vería robótica.
current = Math.floor(easeOut * target)
→ Calculamos el número actual multiplicando el progreso suavizado por el número final, y redondeamos hacia abajo para que sea entero.
requestAnimationFrame(animate)
→ Pedimos al navegador que llame a nuestra función de nuevo en el próximo frame. Esto crea un bucle hasta que la animación termina.
3 El trigger de la animación con x-intersect

Queremos que la animación empiece solo cuando el usuario haga scroll y vea la sección, no antes.

📍 Archivo: resources/views/components/stats-counter.blade.php

<section 
    x-data="statsCounter()"
    x-intersect="startAnimation = true"
>

💡 x-intersect es como un detector de proximidad: detecta cuando el elemento entra en la pantalla.

4 Usar el componente en cualquier página

Una vez creado, usarlo es tan fácil como escribir una etiqueta HTML.

📍 Archivo: Cualquier vista Blade

<!-- Con valores por defecto -->
<x-stats-counter />

<!-- Con valores personalizados -->
<x-stats-counter 
    titulo="Nuestros Números"
    :stats="[
        ['valor' => 5000, 'sufijo' => '+', 'label' => 'Cirugías', 'icono' => 'heart'],
        ['valor' => 50, 'sufijo' => '+', 'label' => 'Doctores', 'icono' => 'users'],
    ]"
/>

💡 Fíjate los dos puntos antes de :stats - eso le dice a Blade que es una expresión PHP, no un string.

📁 ¿Quieres ver el código completo del componente? (opcional)

Esto es solo para que veas dónde va tu código. No necesitas memorizar todo esto:

<!-- Stats Counter - Contador de Estadísticas Animado -->
<!-- 📁 Archivo: resources/views/components/stats-counter.blade.php -->

@props([
    'stats' => [],
    'titulo' => null,
])

@php
$statsDefault = [
    ['valor' => 15000, 'sufijo' => '+', 'label' => 'Pacientes Atendidos', 'icono' => 'users'],
    ['valor' => 15, 'sufijo' => '+', 'label' => 'Años de Experiencia', 'icono' => 'calendar'],
    ['valor' => 25, 'sufijo' => '+', 'label' => 'Especialidades', 'icono' => 'academic-cap'],
    ['valor' => 98, 'sufijo' => '%', 'label' => 'Satisfacción', 'icono' => 'heart'],
];
$stats = !empty($stats) ? $stats : $statsDefault;
@endphp

<section 
    class="py-20 bg-gradient-to-br from-medico-azul-600 to-medico-azul-800"
    x-data="statsCounter()"
    x-intersect="startAnimation = true"
>
    <div class="px-4 mx-auto max-w-7xl">
        <div class="grid grid-cols-2 lg:grid-cols-4 gap-8">
            @foreach($stats as $index => $stat)
                <div class="text-center">
                    <div class="text-4xl md:text-5xl font-bold text-white mb-2">
                        <span x-text="current">0</span>{{ $stat['sufijo'] }}
                    </div>
                    <p class="text-medico-azul-100 text-lg">{{ $stat['label'] }}</p>
                </div>
            @endforeach
        </div>
    </div>

    <script>
        function statsCounter() {
            return { startAnimation: false }
        }
    </script>
</section>

🎉 ¿Qué aprendimos hoy?

  • requestAnimationFrame crea animaciones suaves usando el ciclo de renderizado del navegador
  • La "easing function" (easeOut) hace que las animaciones se vean naturales, no robóticas
  • x-intersect detecta cuando un elemento entra en la pantalla y dispara la animación
  • Los componentes Blade con @props son reutilizables y flexibles
  • El contador animado capta la atención y hace los datos más memorables
