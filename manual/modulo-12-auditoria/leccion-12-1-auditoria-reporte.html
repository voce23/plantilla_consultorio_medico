Error #1

Scopes faltantes en el modelo Especialidad

TipoError en tiempo de ejecución (Call to undefined method)
Archivoapp/Models/Especialidad.php
Dónde explotaCualquier página que tenga el footer — porque footer.blade.php llama a Especialidad::activas()->ordenadas()

🔴 Código roto

❌ ANTES — footer.blade.php usaba estos scopes que no existían 
// En footer.blade.php se llamaba:
$especialidades = Especialidad::activas()->ordenadas()->limit(6)->get();

// Pero en Especialidad.php no existía ni activas() ni ordenadas()

✅ Corrección aplicada

✅ DESPUÉS — app/Models/Especialidad.php 
/**
 * Scope: solo especialidades activas
 */
public function scopeActivas($query)
{
    return $query->where('activo', true);
}

/**
 * Scope: ordenar por nombre
 */
public function scopeOrdenadas($query)
{
    return $query->orderBy('nombre', 'asc');
}
💡 Lección: Cuando un componente llama a un scope de Eloquent, ese scope DEBE existir en el modelo. Si no existe, el sitio explota en tiempo de ejecución.

Error #3

Ruta blog.categoria no registrada

TipoRouteNotFoundException al cargar un artículo del blog
Quién la llamaresources/views/articulos/show.blade.php — línea 22
CorrecciónRuta agregada en routes/web.php
✅ routes/web.php — ruta agregada 
// Ruta que faltaba — usada en articulos/show.blade.php
Route::get('/blog/categoria/{categoria}', function (string $categoria) {
    return view('paginas.blog', ['categoria' => $categoria]);
})->name('blog.categoria');
⚠️ Nota importante: Esta ruta debe ir ANTES de /blog/{slug} en web.php para que Laravel no confunda "categoria" como un slug. Está posicionada correctamente.
💡 Lección: Siempre que creas una vista que usa route(), verifica que esa ruta exista en web.php. Una forma rápida es ejecutar php artisan route:list y buscar el nombre.
